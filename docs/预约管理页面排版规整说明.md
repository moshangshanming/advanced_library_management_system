## 设计原则

### 1. 模块化设计
- 每个功能区域都有明确的边界和分隔
- 模块间使用细横线（`border-bottom: 1px solid var(--border)`）分隔
- 保持视觉层次清晰

### 2. 统一性原则
- **控件高度**：所有输入框、下拉框、按钮高度统一为44px
- **间距统一**：模块间padding统一为16-20px
- **字体大小**：保持一致的字体大小规范
- **颜色规范**：使用系统定义的CSS变量，确保颜色一致性

### 3. 响应式布局
- 查询筛选区使用 `flex-wrap: nowrap` 确保单行显示
- 分页控制区使用 `justify-content: space-between` 实现左右对齐
- 所有控件都设置了合理的min-width和max-width

### 4. 视觉层次
- 标题区：最醒目，使用深色字体
- 规则说明区：中性灰色，不抢视觉焦点
- 查询筛选区：清晰的控件排列，便于操作
- 操作按钮区：表格主体，占据主要视觉空间
- 分页控制区：底部对齐，方便导航

## 兼容性说明

- 所有修改均基于现有的CSS变量系统
- 使用了标准的Flexbox布局，兼容现代浏览器
- 保持了与系统其他页面的一致性
- 不影响现有功能逻辑
